# 架构说明

## 设计目标

这次重写采用“核心无 UI，外层负责集成”的思路：

- core 只负责文本缓冲区、光标、撤销重做，不依赖 gg 或文件系统
- editor 只处理模式和命令映射，把键盘事件转成可验证的状态变更
- editor/session.v 负责多缓冲区会话管理，前端不直接管理标签切换规则
- infra 提供文件系统能力，避免编辑核心直接触盘
- infra/workspace.v 负责工作区文件索引和纯过滤逻辑
- frontend 负责 gg 窗口、绘制和事件桥接
- app 负责启动期装配，而不是承载运行期全部状态

## 分层关系

```text
main -> app -> frontend -> editor -> core
                 |           \
                 |            -> infra
                 -> infra
```

## 与参考项目相比的关键改进

1. 不再使用单个巨型 Ved 结构体承载所有运行时状态。
2. Buffer 可单独测试，不需要窗口环境即可验证编辑行为。
3. 多标签切换由 Session 处理，前端只消费当前状态和标签标题。
4. 输入分发收敛到 editor.State，后续可替换成更完整的命令系统。
5. 前端只关心渲染和事件桥接，未来替换成 TUI 或其他图形后端时成本更低。

## 当前阶段能力边界

这是一个第一阶段 MVP，而不是完整功能对等版本。

已完成：

- 单缓冲区编辑
- 多标签会话与切换
- 工作区文件索引与 Ctrl+P 打开
- 基础 Vim 风格按键
- 保存 / 退出 / 撤销 / 重做
- 原生窗口渲染

未完成：

- 多工作区切换
- 多分屏
- 模糊搜索
- 语法高亮
- git 集成
- 调试器
- 补全

## 扩展建议

下一步可以在当前结构上继续补充：

- core/history.v: 把快照式历史替换成增量式操作日志
- editor/commands.v: 引入显式 Command 枚举和 dispatcher
- frontend/layout.v: 把状态栏、编辑区、面板布局进一步拆分
- infra/workspace.v: 提供目录索引和缓存
